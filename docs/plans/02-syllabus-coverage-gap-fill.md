# 02. 공식 출제범위 커버리지 갭 보완

**날짜**: 2026-07-13
**배경**: 사용자가 "전체적으로 내용을 점검하여 시험 준비에 부족한 부분이 없는지 확인"을 요청했다. 8개 챕터(1과목 3개, 2과목 2개, 3과목 3개)의 이론·문제·모의고사를 3개 과목으로 나누어 병렬 Explore 에이전트로 실제 ADsP 공식 출제범위 체크리스트와 대조 점검했다.

## 발견된 갭

### 완전히 빠진 토픽 (이론+문제 모두 없음)
1. **데이터베이스 정의 및 4대 특징** (1과목 1장) — 공유/통합/저장/운영(변화) 데이터
2. **빅데이터 가치 측정이 어려운 이유** (1과목 2장) — 재사용/다목적활용/재조합
3. **확률의 기본개념** (3과목 2장) — 확률 정의, 덧셈정리, 조건부확률, 독립사건
4. **데이터웨어하우스(DW) Inmon 4대 특성** (3과목 1장) — 주제지향적/통합성/시계열성/비휘발성

### 이론/문항 불일치
5. **나선형 모델(Spiral Model)** (2과목 2장) — `part2_ch2.json`의 `p2c2_001` 해설이 이미 사용하는데 이론에는 전혀 없던 용어. 기존 "반복적 정련" 설명에 정식 명칭을 붙여 해결.

### 명칭만 있고 세부 내용이 얕음
6. **기술통계 세부 종류** (3과목 2장) — 중심경향값(평균/중앙값/최빈값), 특히 **변동계수(CV)**가 이름조차 없었음
7. **분석 성숙도 3개 진단부문** (2과목 2장) — 비즈니스/조직·역량/IT 부문 명칭만 있고 평가 내용 없음
8. **SEMMA 각 단계 세부 설명** (2과목 1장) — "(참고)"로 단계명만 나열

### 콘텐츠 볼륨 불균형
9. `part3_ch1.json` 문제은행이 3문항뿐 — 다른 7개 챕터(15~22문항) 대비 크게 부족

## 수정 내역

### 이론(.md) — 6개 파일
- `part1_ch1.md`: "데이터베이스의 이해" 섹션(정의 + 4대 특징 표) 신설
- `part1_ch2.md`: "빅데이터의 가치 산정이 어려운 이유" 섹션 신설, "Mckinseny" → "McKinsey" 오탈자 수정
- `part2_ch1.md`: SEMMA 절을 KDD/CRISP-DM 수준으로 5단계 상세 설명으로 확장
- `part2_ch2.md`: "나선형 모델(Spiral Model)" 명칭을 반복적 정련 설명에 결합, 분석성숙도 3개 진단부문(비즈니스/조직·역량/IT) 평가 내용 표 추가
- `part3_ch1.md`: "데이터 웨어하우스(DW)의 4대 특성" 섹션(Inmon 4대 특성 표) 신설
- `part3_ch2.md`: "확률의 기본개념" 섹션 신설(확률 정의/덧셈정리/조건부확률/독립사건), 기술통계 절을 중심경향값(평균/중앙값/최빈값)·산포도(분산/표준편차/IQR/**변동계수**)·왜도·첨도 해석으로 구체화

### 문제(.json) — 6개 파일, 총 24문항 신규 추가
| 파일 | 신규 문항 | 비고 |
|---|---|---|
| `part1_ch1.json` | 2개 | 데이터베이스 4대 특징 |
| `part1_ch2.json` | 2개 | 가치 산정이 어려운 이유 |
| `part2_ch1.json` | 1개 | SEMMA 단계별 활동 |
| `part2_ch2.json` | 2개 | 나선형 모델, 성숙도 3개 진단부문 |
| `part3_ch1.json` | 12개 | DW 4대특성 2개 + 결측값(MAR/MNAR/완전분석법/다중대치법) + 데이터변환(정규화/표준화/구간화) + 파생·요약변수 + ETL 등, 3문항→15문항으로 확충 |
| `part3_ch2.json` | 5개 | 확률기초(덧셈정리/조건부확률/독립사건) 3개 + 변동계수·중앙값 2개 |

### 메타데이터 동기화
- `lib/chapters.ts`의 `questionCount` 8곳을 최종 실측치로 갱신 (17/22/20/21/24/15/25/21)
- `README.md`, `CLAUDE.md`의 "문제 현황" 표를 최신 총합(챕터 165문항 + 모의고사 100문항 = 총 265문항)으로 갱신
- README.md "알려진 제한사항"에서 `part3_ch1` 문항 부족 항목 제거 (해소됨)

## 명시적으로 범위 제외한 항목 (참고용)
- 오류가 아닌 경미한 구조적 중복(1과목 2·3장의 데이터사이언스 영역 서술 중복) — 통합 작업 안 함
- 3과목 3장의 KDD/SEMMA 상호참조, ANOVA 사후검정 — 선택사항으로 보류
- 2과목 1장 Data Size/Speed를 정답으로 하는 문항 편중 — 우선순위 낮아 보류
- 모의고사(exam1/exam2)에는 이번 신규 토픽을 반영하지 않음 — 모의고사는 이미 배점 비율·R배제 원칙에 맞춰 완성된 상태이므로 별도 검토 필요

## 검증
- `npx tsx scripts/validate-questions.ts` → 파일 10개, 문제 265개, 고유 ID 265개, 오류 0건
- `lib/chapters.ts`의 questionCount 8개 값이 실제 JSON 배열 길이와 전부 일치 (자동 스크립트로 재확인)
- `npm run type-check`, `npm run test`(13개), `npm run build`(SSG 8개 이론 페이지 정상 렌더링) 모두 통과
