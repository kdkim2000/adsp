# 3과목 3장: 정형 데이터 마이닝

## 1. 데이터 마이닝(Data Mining)의 개요

### 1) 정의 및 통계분석과의 비교

* **정의**: 대용량 데이터로부터 이전에는 알지 못했던 유용한 패턴, 규칙, 지식을 발견하여 비즈니스 의사결정에 활용하는 일련의 과정이다.
* **통계분석과의 차이점**
  * **통계분석**: 사전에 수립된 가설이나 가정을 바탕으로 표본(Sample) 데이터를 추출하여 가설의 유의성을 검증하는 하향식(Top-down) 접근 방식. 엄격한 수리적 모델링과 정형화된 정량 데이터 중심이다.
  * **데이터 마이닝**: 명확한 가설 없이 다양한 수리적 알고리즘을 이용해 방대한 데이터 자체에서 의미 있는 정보와 감춰진 패턴을 스스로 찾아내는 상향식(Bottom-up) 접근 방식. 정형 데이터뿐만 아니라 반정형·비정형 데이터까지 총체적으로 다룬다.

### 2) 분석 목적 및 데이터 형태에 따른 분류

* **지도학습(Supervised Learning)**: 데이터에 명확한 정답(레이블, Label)이 존재하는 상황에서 학습하는 방법
  * **분류(Classification)**: 종속변수(타깃)가 범주형(이산형)인 경우 (예: 스팸 여부 0/1, 신용 등급 A/B/C)
  * **예측·추정(Prediction/Estimation)**: 종속변수(타깃)가 연속형 숫자인 경우 (예: 내일의 주가, 다음 달 매출액)
* **비지도학습(Unsupervised Learning)**: 데이터에 정답(레이블)이 존재하지 않고 독립변수만 있는 상태에서 데이터 자체의 결합도나 구조를 탐색하는 방법
  * **군집 분석(Clustering)**: 개체 간의 유사성을 바탕으로 유사한 그룹끼리 묶어주는 기법
  * **연관 분석(Association Rule)**: 품목 간의 동시 발생 확률을 분석하여 규칙을 발견하는 기법 (장바구니 분석)

### 3) 데이터 마이닝 추진 단계 (CRISP-DM 방법론 기준)

데이터 마이닝은 일반적으로 글로벌 표준 방법론인 CRISP-DM(CRoss Industry Standard Process for Data Mining)의 6단계 프로세스에 따라 수행된다.

1. **비즈니스 이해(Business Understanding)**: 비즈니스 관점의 목적과 요구사항 파악, 프로젝트 정의 및 초기 계획 수립
2. **데이터 이해(Data Understanding)**: 초기 데이터 수집, 데이터 특성 파악, 데이터 탐색(EDA), 데이터 품질 문제 확인
3. **데이터 준비(Data Preparation)**: 수집된 원천 데이터로부터 분석용 데이터셋을 편성하는 단계로, 전체 과정 중 가장 많은 시간과 비용(약 60~80%)이 소요됨. 결측치 처리, 이상치 제거, 데이터 정제·통합·변환, 파생변수 생성 등을 수행
4. **모델링(Modeling)**: 다양한 데이터 마이닝 알고리즘(의사결정나무, 인공신경망 등)을 선택하고 최적의 파라미터를 튜닝하여 모형을 생성. 학습·검증·테스트 데이터 분할도 이 단계에서 함께 이루어짐
5. **평가(Evaluation)**: 생성된 모델이 비즈니스 목적 및 목표에 부합하는지 검증
6. **전개(Deployment)**: 완성된 모형을 실무 운영 시스템에 적용하고, 모니터링 및 유지보수 계획을 수립하여 종료 보고서 작성

## 2. 분류 분석 (Classification)

분류 분석은 개체를 정해진 몇 개의 범주(Class) 중 하나로 분류하는 모형을 설계하는 기법이다.

### 1) 의사결정나무 (Decision Tree)

* **특징**
  * 나무 구조 형태로 분류 규칙을 표현하여, 분석 과정 및 결과를 사용자가 쉽게 이해할 수 있는 해석 용이성과 시각화가 최대 장점이다.
  * 대용량 데이터 처리가 가능하고, 데이터 성격(연속형/범주형)에 구애받지 않으며 상관성이 높은 변수가 있어도 모형이 비교적 안정적이다.
  * 연속형 변수를 불연속적인 값으로 취급하므로 예측 경계면에서 오차가 클 수 있고, 대형 나무 모형은 변수의 작은 변화에도 구조가 급변하여 과적합(Overfitting)에 취약하다.

* **나무의 형성 과정**
  1. **성장(Growth)**: 각 마디에서 적절한 분리 기준(Splitting Criterion)에 따라 자식 마디를 형성하며 트리를 키운다. 정지 규칙(Stopping Rule)을 충족하면 성장을 멈춘다.
  2. **가지치기(Pruning)**: 지나치게 비대해진 나무 모형은 학습 데이터에만 맞춰져 일반화 능력이 떨어진다. 불필요한 마디나 가지를 제거하여 모형을 단순화하고 과적합을 방지한다.
  3. **타당성 평가**: 이익도표(Gain Chart), 위험명세서 등을 활용해 최종 나무를 평가한다.

* **분리 기준 지표**: 부모 노드보다 자식 노드의 불순도(Impurity)가 감소하는(순도가 증가하는) 방향, 즉 정보획득량(Information Gain)이 최대가 되는 방향으로 분할 변수와 분할점을 선택한다.

  * **범주형(이산형) 타깃 변수의 분리 기준**
    * **지니 지수(Gini Index)**:
      ```
      Gini(T) = 1 - Σ pᵢ²   (i = 1 ... k, pᵢ = 각 범주의 비율)
      ```
      값이 **작을수록** 노드 내 순도가 높고 잘 분리된 것이다. **CART 알고리즘**에서 사용한다.
    * **엔트로피 지수(Entropy Index)**:
      ```
      Entropy(T) = - Σ pᵢ log₂(pᵢ)
      ```
      값이 0일 때 가장 순수하며, **C4.5 / C5.0 알고리즘**에서 사용한다.
    * **카이제곱 통계량의 p-value**: p-value가 가장 작은(유의성이 가장 높은) 변수를 선택하여 분리한다. **CHAID 알고리즘**에서 사용한다.
  * **연속형 타깃 변수의 분리 기준**
    * **F-통계량**: 일원배치 분산분석의 F-통계량 p-value가 가장 작은 변수 기준 (CHAID)
    * **분산의 감소량(Variance Reduction)**: 자식 노드의 분산 합이 최소가 되도록 분리 (CART)

* **주요 알고리즘의 R 함수**
  * `rpart` 패키지의 `rpart()` 함수: CART 알고리즘 기반 (기본 지니 지수 사용)
  * `party` 패키지의 `ctree()` 함수: 조건부 추론 나무(Conditional Inference Tree) 알고리즘 기반으로, 다중 가설 검정에 기반한 p-value를 사용하여 별도의 가지치기가 필요 없다는 특징이 있다.

### 2) 로지스틱 회귀분석 (Logistic Regression)

* **개념**: 종속변수가 범주형(주로 이진형, 성공 1 또는 실패 0)일 때 독립변수와 종속변수 간의 관계를 설명하고 분류하는 회귀 기반 모형이다. 일반 선형 회귀와 달리 최소제곱법이 아닌 **최대우도추정법**(MLE)으로 파라미터를 추정한다.
* **핵심 원리 및 변환 과정**: 확률 p 자체를 회귀식의 종속변수로 두면 오른쪽 항의 범위(-∞ ~ +∞)와 왼쪽 항의 범위(0 ≤ p ≤ 1)가 일치하지 않는 문제가 발생하므로 변환을 거친다.
  * **오즈(Odds)**: 실패 확률 대비 성공 확률의 비
    ```
    Odds = p / (1 - p)      범위: 0 ~ +∞
    ```
  * **로짓 변환(Logit Transformation)**: 오즈에 자연로그를 취해 종속변수의 범위를 -∞ ~ +∞로 완전히 일치시키는 과정
    ```
    ln( p / (1-p) ) = β₀ + β₁X₁ + β₂X₂ + ... + βₖXₖ
    ```
  * **시그모이드 함수(Sigmoid/Logistic Function)**: 로짓 변환된 선형 회귀식을 다시 확률 p에 대해 정리한 공식으로, S자 형태의 누적분포함수 모양을 가진다.
    ```
    p = 1 / (1 + e^-(β₀+β₁X₁+...+βₖXₖ))
    ```
* **해석 방법(Odds Ratio)**: 독립변수 X₁이 1단위 증가할 때, 다른 독립변수들이 고정되어 있다면 성공의 **오즈(Odds)가 e^β₁배 증가**한다.
* **R 함수**: `glm(formula, data, family = "binomial")` 형태로 family 인자에 반드시 `"binomial"`을 지정해야 로지스틱 회귀분석이 수행된다.

> **핵심**: 로지스틱 회귀의 계수 β₁을 해석할 때 "확률이 e^β₁배 증가한다"가 아니라 "**오즈**가 e^β₁배 증가한다"이다. 확률과 오즈를 혼동하는 오답에 주의한다.

### 3) 인공신경망 (Artificial Neural Network, ANN)

* **구조**: 인간 뇌의 뉴런 구조를 모방
  * **입력층(Input Layer)**: 외부의 설명변수 값들을 받아들이는 층. 변수의 개수만큼 노드가 존재
  * **은닉층(Hidden Layer)**: 입력층으로부터 들어온 신호들의 가중합을 계산하고 활성화 함수를 통해 비선형 변환을 가하여 복잡한 패턴을 추출하는 층
  * **출력층(Output Layer)**: 최종 예측 타깃 값을 도출하는 층

* **핵심 메커니즘**
  * **순전파(Forward Propagation)**: 입력층에서 은닉층을 거쳐 출력층 방향으로 신호가 전달되어 예측값이 계산되는 과정
  * **역전파(Backpropagation)**: 출력층에서 발생한 예측 오차(Error)를 역방향(출력층 → 은닉층 → 입력층)으로 전파시키며 가중치(Weight)와 편향(Bias)을 점진적으로 수정·최적화하는 경사하강법 기반 학습 알고리즘

* **활성화 함수(Activation Function)**: 입력 노드들의 가중합을 최종 출력 신호로 변환하는 함수로, 신경망 모형에 비선형성(Non-linear)을 부여하는 핵심 장치이다.
  * **계단 함수(Step Function)**: 임계값을 넘으면 1, 넘지 못하면 0을 반환하는 불연속 함수
  * **시그모이드 함수**: 출력을 0과 1 사이의 연속적인 확률값으로 반환. 층이 깊어질수록 경사도가 0으로 수렴하는 **기울기 소실(Vanishing Gradient) 문제**가 발생함
  * **하이퍼볼릭 탄젠트(tanh)**: 출력을 -1과 1 사이의 값으로 변환. 시그모이드보다 중심값이 0에 가깝지만 여전히 기울기 소실 문제가 존재함
  * **ReLU 함수(Rectified Linear Unit)**: 입력이 0 이하면 0을, 0을 초과하면 입력값 그대로를 반환. 계산이 빠르고 기울기 소실 문제를 극복한 가장 대중적인 함수
  * **소프트맥스 함수(Softmax Function)**: 출력층에서 주로 사용되며, 모든 출력 노드 값의 합이 1이 되도록 정규화하여 각 클래스에 속할 사후 확률을 제공. **다중 분류(Multi-class Classification)** 모형의 최종 출력단에 필수적으로 사용

* **신경망 통제 고려사항**: 은닉층의 노드 수가 너무 많으면 과적합(Overfitting), 너무 적으면 과소적합(Underfitting)이 발생한다. 과적합 방지를 위해 가중치를 감소시키는 **가중치 감쇠**(Weight Decay)나 임의의 노드를 생략하는 **드롭아웃(Dropout)** 기법을 적용한다.

### 4) 서포트 벡터 머신 (Support Vector Machine, SVM)

* **개념**: 데이터를 분리하는 수많은 초평면 중에서 두 클래스 간의 거리를 가장 크게 넓혀주는 **최적 분리 초평면**(Optimal Separating Hyperplane)을 찾는 기법이다.
* **핵심 용어**
  * **마진(Margin)**: 결정 경계(초평면)에서부터 가장 가까운 데이터 포인트까지의 거리. SVM은 이 마진을 최대화하는 것을 목표로 한다.
  * **서포트 벡터(Support Vector)**: 결정 경계를 결정하는 데 있어 직접적인 기준점이 되는, 초평면에 가장 가까이 붙어 있는 핵심 데이터 포인트들
  * **슬랙 변수(Slack Variable)**: 선형 분류가 완벽히 불가능한 상황에서 약간의 오분류를 허용하는 정도를 조절하기 위해 도입된 파라미터
  * **커널 트릭(Kernel Trick)**: 저차원에서 선형 분리가 불가능한 데이터를 고차원 공간(Feature Space)으로 매핑하여 선형 분리가 가능하도록 만드는 기법. 다항식(Polynomial) 커널, 방사기저함수(RBF/Gaussian) 커널 등이 대표적이다.

### 5) 앙상블 기법 (Ensemble)

단일 모델의 한계를 극복하기 위해 여러 개의 기초 분류 모형(Weak Learner)을 생성하고 그 예측 결과들을 결합(투표 또는 평균)하여 최종 예측 성능과 안정성을 높이는 방법론이다.

* **배깅(Bagging, Bootstrap Aggregating)**
  * **원리**: 원본 데이터셋에서 크기가 동일한 대상을 무작위로 복원 추출하여 여러 개의 **부트스트랩(Bootstrap)** 샘플을 생성한다. 각 샘플마다 독립적인 모델을 병렬적으로 학습시킨 후, 분류 문제는 다수결 투표(Voting), 회귀 문제는 평균(Aggregation)으로 결과를 결합한다.
  * **부트스트랩의 특성**: 복원 추출 방식이므로 전체 데이터 중 약 **63.2**%만 샘플에 포함되고, 나머지 약 36.8%는 한 번도 추출되지 않는데 이를 **OOB(Out-Of-Bag)** 데이터라 부르며 모델 검증용으로 활용한다.
  * **효과**: 모델의 **분산(Variance)을 감소**시켜 과적합을 방지하는 데 효과적이다.

* **부스팅(Boosting)**
  * **원리**: 이전 단계 모형이 잘못 분류한(오차가 큰) 데이터에 **가중치를 높게 부여**하여, 다음 단계 모형이 해당 오차 데이터를 집중적으로 학습하도록 모형을 순차적으로 연결·누적한다.
  * **효과**: 모델의 **편향(Bias)을 감소**시키는 데 초점을 둔다. 성능이 우수하나 이상치(Outlier)에 민감할 수 있다.
  * **대표 알고리즘**: AdaBoost, Gradient Boosting Machine(GBM), XGBoost, LightGBM 등

* **랜덤 포레스트(Random Forest)**
  * **원리**: 배깅에 설명변수의 무작위 선택을 추가한 알고리즘이다. 나무의 각 노드를 분할할 때마다 전체 변수 중 일부만 무작위로 선택하여 분할 기준으로 활용한다.
  * **효과**: 나무들 간의 상관관계를 줄여 단일 배깅 나무 모형보다 분산 감소 효과를 극대화하고 과적합을 방지한다. 변수 중요도(Variable Importance)를 제공한다는 실무적 장점이 있다.

> **핵심**: 배깅은 병렬적으로 학습하며 분산을 줄이는 데, 부스팅은 순차적으로 학습하며 편향을 줄이는 데 초점을 둔다는 차이를 구분해야 한다.

## 3. 군집 분석 (Clustering)

군집 분석은 개체들의 유사성을 측정하여 사전에 정의된 범주(레이블)가 없는 상태에서 유사성이 높은 개체들끼리 몇 개의 군집으로 집단화하는 대표적인 비지도학습 기법이다.

### 1) 데이터 간 거리 측정 방법

군집화를 하려면 개체 간 거리를 정의해야 하며, 변수의 속성에 따라 거리 측정법이 다르다.

* **연속형 변수의 거리**
  * **유클리드 거리(Euclidean Distance)**: 가장 일반적인 기하학적 최단 거리. `d(x,y) = √(Σ(xᵢ-yᵢ)²)`
  * **맨하탄 거리(Manhattan Distance)**: 격자 모양의 도로를 따라 이동할 때의 거리. `d(x,y) = Σ|xᵢ-yᵢ|`
  * **민코프스키 거리(Minkowski Distance)**: 유클리드와 맨하탄을 일반화한 거리(Lp Norm). `d(x,y) = (Σ|xᵢ-yᵢ|^p)^(1/p)` — p=1이면 맨하탄, p=2이면 유클리드 거리와 동일
  * **마할라노비스 거리(Mahalanobis Distance)**: 변수 간의 상관관계(공분산)를 고려하여 확률분포를 반영한 통계적 거리
  * **체비셰프 거리(Chebyshev Distance)**: 변수 간 거리 차이의 최댓값을 기준으로 측정

* **범주형(이진형) 변수의 거리**: 두 개체의 이진 속성을 아래처럼 정리한 뒤 계산한다 — a: 둘 다 1, b: 개체X만 1, c: 개체Y만 1, d: 둘 다 0
  * **자카드 계수(Jaccard Coefficient)**: 둘 다 0인 경우(d)는 의미가 없다고 보고 제외. `Jaccard = a / (a+b+c)`
  * **단순 일치 계수(Simple Matching Coefficient)**: 전체 속성 중 일치하는 빈도. `SMC = (a+d) / (a+b+c+d)`

### 2) 계층적 군집분석 (Hierarchical Clustering)

* **특징**: 개별 개체들을 순차적으로 가장 유사한 것부터 묶어 올라가는 방식으로, 사전에 군집 수(K)를 지정할 필요가 없다. 군집화 과정을 트리 구조의 **덴드로그램**(Dendrogram)으로 시각화할 수 있다. 한 번 군집에 속하면 다른 군집으로 다시 이동할 수 없다.
* **군집 간 거리 연결법**
  * **최단 연결법(Single Linkage)**: 두 군집의 관측치 쌍 간 거리 중 최솟값. 사슬 효과(Chaining Effect)로 길게 늘어진 비정형 군집이 형성될 우려가 있음
  * **최장 연결법(Complete Linkage)**: 두 군집의 관측치 쌍 간 거리 중 최댓값. 내부 결합도가 조밀한 군집 형성
  * **평균 연결법(Average Linkage)**: 두 군집의 모든 관측치 쌍 간 거리의 산술 평균
  * **중심 연결법(Centroid Linkage)**: 두 군집의 중심점(평균) 사이의 거리
  * **와드 연결법(Ward's Method)**: 군집 간 거리를 직접 재지 않고, 두 군집이 결합함으로써 발생하는 군집 내 오차제곱합(SSE)의 증가량을 최소화하는 방향으로 병합. 크기가 비슷한 조밀한 구형 군집을 형성하는 경향이 있음

### 3) 비계층적 군집분석: K-평균 (K-Means Clustering)

* **특징**: 군집의 개수(K)를 분석가가 사전에 지정해야 한다. 대용량 데이터셋에 신속하게 적용 가능하지만, 거리 기반이므로 이상치와 초기 중심점 위치에 민감하며 볼록한 구형(Spherical) 군집만 잘 찾아낸다.
* **작동 프로세스**
  1. 분석가가 지정한 K개의 초기 중심점(Centroid)을 임의로 배치
  2. 모든 데이터 포인트를 가장 가까운 중심점에 할당하여 K개의 임시 군집 형성
  3. 각 군집에 할당된 데이터의 평균값으로 중심점을 새롭게 갱신
  4. 중심점 위치 변화가 없을 때(수렴)까지, 또는 최대 반복 횟수에 도달할 때까지 2~3번을 반복

### 4) 혼합 분포 군집분석 (Mixture Distribution Clustering)

* **개념**: 데이터가 여러 개의 연속형 확률분포(주로 다변량 정규분포)의 혼합(Mixture)으로부터 생성되었다고 가정하고 분포의 파라미터를 역추적하는 확률 기반 군집화 기법이다.
* **EM 알고리즘(Expectation-Maximization)**: 각 정규분포의 평균·분산 파라미터의 최대우도추정량(MLE)을 구하는 반복 알고리즘
  * **E-단계(Expectation)**: 현재 파라미터를 기반으로 각 데이터가 특정 군집에 속할 사후 확률(기대치)을 계산
  * **M-단계(Maximization)**: E-단계에서 구한 확률을 가중치로 삼아 우도(Likelihood)를 최대화하도록 파라미터를 업데이트. 수렴할 때까지 E·M 단계를 반복
* **특징**: K-평균과 달리 개체가 특정 군집에 속할 확률을 소프트하게 제공(Soft Assignment)하며 비원형 군집도 유연하게 찾아내지만, 데이터가 적으면 파라미터 추정이 어렵고 계산량이 크다.

### 5) SOM (Self-Organizing Map, 자기조직화지도)

* **개념**: 고차원의 다변량 데이터를 2차원의 격자(Grid) 평면 구조로 축소하여 시각화와 군집화를 동시에 수행하는 인공신경망 기반 기법이다.
* **구조**: 입력층과 경쟁층(Competitive Layer, 출력층)의 두 개 층으로만 구성된 전방향(Feed-forward) 신경망 구조이다 (은닉층 없음). 입력층의 각 노드는 설명변수의 개수만큼 존재하며, 경쟁층의 뉴런들과 완전히 연결되어 있다.
* **경쟁 학습(Competitive Learning)**: 입력 패턴이 들어오면 경쟁층 뉴런 중 유클리드 거리가 가장 가까운 뉴런이 승리하며, 이 승자 뉴런을 **BMU**(Best Matching Unit)라 부른다. 승자 뉴런과 그 주변 이웃 뉴런들의 가중치만 업데이트하는 승자 독식(Winner-take-all) 방식을 취한다.

### 6) 군집 성능 평가지표: 실루엣 계수 (Silhouette Coefficient)

* **정의**: 군집화 완료 후, 개체들이 군집 내에서 얼마나 응집되어 있고 서로 다른 군집 간에는 얼마나 명확히 분리되어 있는지를 동시에 측정하는 지표
* **산식**: `s(i) = (b(i) - a(i)) / max(a(i), b(i))`
  * a(i): 개체 i와 동일 군집 내 다른 개체들 간의 평균 거리
  * b(i): 개체 i와 가장 가까운 이웃 군집에 속한 개체들 간의 평균 거리
  * 범위: -1 ≤ s(i) ≤ 1
* **해석**: 1에 가까울수록 군집화가 잘 이루어진 것(일반적으로 평균 0.5 이상이면 타당), 0에 가까울수록 군집 간 경계가 모호함, 음수에 가까울수록 개체가 잘못된 군집에 할당되었음을 의미한다.

## 4. 연관 분석 (Association Rule Analysis)

연관 분석은 대규모 거래 기록(Transaction Data) 속에서 "A 품목을 구매한 고객은 B 품목도 함께 구매한다"와 같은 연관 규칙을 발견하는 기법이다 (장바구니 분석).

### 1) 연관성 측정 3대 평가지표

* **지지도(Support)**: 전체 거래 건수 중 품목 A와 B가 동시에 포함되어 구매된 거래의 비율. 규칙의 전반적인 빈도를 나타낸다.
  ```
  Support(A→B) = P(A∩B) = (A와 B가 동시에 포함된 거래 수) / (전체 거래 수)
  ```
* **신뢰도(Confidence)**: 품목 A를 구매한 거래 중에서 품목 B도 함께 구매했을 조건부 확률. 규칙의 확실성(강도)을 나타낸다.
  ```
  Confidence(A→B) = P(B|A) = Support(A→B) / Support(A)
  ```
* **향상도(Lift)**: 품목 A의 구매 여부가 품목 B의 구매 확률을 우연 수준에 비해 얼마나 증가시키는지 측정
  ```
  Lift(A→B) = Confidence(A→B) / Support(B) = P(A∩B) / (P(A)×P(B))
  ```
  * **Lift = 1**: A와 B는 서로 아무 상관관계가 없는 독립적인 관계 (연관 규칙으로서 가치 없음)
  * **Lift > 1**: 양의 상관관계. A를 구매한 것이 B를 구매할 확률을 높이는 유의미한 연관 규칙
  * **Lift < 1**: 음의 상관관계. A를 구매하면 오히려 B를 구매하지 않을 확률이 높아지는 대체재 성격

### 2) Apriori 알고리즘의 원리

* 항목의 개수가 많아질수록 가능한 연관 규칙의 가짓수가 기하급수적으로 증가하는 문제를 해결하기 위한 알고리즘이다.
* **최소 지지도**(Minimum Support)를 기준으로 규칙을 탐색한다.
* **가지치기 원리**: "한 항목집합이 빈번(Frequent)하지 않다면, 그 항목집합을 포함하는 모든 상위 항목집합 또한 빈번할 리 없다"는 원리를 활용하여 불필요한 탐색 공간을 줄여 계산 효율성을 높인다.

### 3) R 분석 패키지 및 함수

* 연관 분석에는 `arules` 패키지를 사용하며, 함수명은 `apriori(data, parameter = list(support = 0.1, confidence = 0.5))` 형태이다.
* `apriori()` 함수로 도출된 연관 규칙 결과를 텍스트 형태로 확인하려면 `inspect()` 함수를 호출해야 한다.

## 5. 분류 모형 평가 (Model Evaluation)

### 1) 오분류표 (Confusion Matrix, 혼동 행렬)

모형이 예측한 값과 실제 데이터의 정답 범주를 매칭하여 빈도를 표로 나타낸 것이다.

| 구분 | 예측 양성(Predicted Positive) | 예측 음성(Predicted Negative) |
|---|---|---|
| **실제 양성(Actual Positive)** | TP (True Positive) | FN (False Negative) |
| **실제 음성(Actual Negative)** | FP (False Positive) | TN (True Negative) |

* **제1종 오류**: 실제 음성인 대상을 모형이 양성이라고 잘못 예측한 오류 (→ FP)
* **제2종 오류**: 실제 양성인 대상을 모형이 음성이라고 잘못 예측한 오류 (→ FN)

### 2) 평가지표 수식 요약

| 지표 | 산식 | 의미 |
|---|---|---|
| **정확도(Accuracy)** | (TP+TN) / (TP+TN+FP+FN) | 전체 중 올바르게 분류한 비율 |
| **정밀도(Precision)** | TP / (TP+FP) | 양성으로 예측한 것 중 실제 양성 비율 |
| **재현율(Recall)/민감도(Sensitivity)** | TP / (TP+FN) | 실제 양성 중 양성으로 올바르게 예측한 비율 |
| **특이도(Specificity)** | TN / (TN+FP) | 실제 음성 중 음성으로 올바르게 예측한 비율 |
| **위양성률(FPR)** | FP / (TN+FP) = 1 - 특이도 | 실제 음성을 양성으로 잘못 예측한 비율 |
| **F1-Score** | 2 × (Precision×Recall) / (Precision+Recall) | 정밀도와 재현율의 조화평균 |

> **핵심**: 암 진단, 금융 사기 적발, 결함 탐지처럼 실제 양성을 음성으로 놓쳤을 때(FN) 치명적인 리스크가 발생하는 도메인에서는 정확도보다 **재현율**(Recall)을 우선 지표로 관리해야 한다.

### 3) ROC 곡선과 AUC (Area Under Curve)

* **ROC 곡선**: 세로축에 민감도(TPR = 재현율), 가로축에 위양성률(FPR = 1-특이도)을 두고 분류 임계값(Threshold)을 0~1까지 변화시키며 모델의 분류 판별력을 시각화한 곡선
* **해석**: 모형의 판별 성능이 완벽할수록 ROC 곡선은 좌측 상단 꼭짓점 (0, 1)에 가까워진다. 분별력이 없는 무작위 모형은 (0,0)과 (1,1)을 잇는 45도 대각선 형태를 띤다.
* **AUC**: ROC 곡선 아래의 면적. 최소 0.5(무작위 모형)에서 최대 1.0(완벽한 모형) 사이의 범위를 가지며, 1에 가까울수록 판별력이 우수한 모형이다.

### 4) 향상도 이익도표 (Gain / Lift Chart)

모형을 적용하지 않고 무작위로 타깃을 선택했을 때의 반응률과, 모형이 예측한 확률이 높은 순으로 정렬하여 상위 구간(주로 10분위)을 선택했을 때의 반응률을 비교하여 향상 정도를 나타내는 도표이다.

```
Lift Factor = (해당 구간 모형 적용 시 반응률) / (무작위 선택 시 기본 반응률)
```

모형의 성능이 우수할수록 상위 구간(10~20%)에서 향상도가 급격히 높게 나타나다가 하위 구간으로 갈수록 낮아진다. 성능이 나쁜 모형은 전 구간에서 향상도가 1에 가깝게 평평하게 유지된다.

### 5) 교차 검증 (Cross Validation)

* 데이터가 부족할 때 발생할 수 있는 평가 오차를 줄이고 과적합을 방지하기 위해 데이터를 분할하여 학습과 평가를 반복하는 기법이다.
* **K-Fold 교차 검증**: 데이터를 K개의 동일한 크기 폴드(Fold)로 나눈 후, K-1개는 학습 데이터로, 나머지 1개는 평가 데이터로 사용하는 과정을 K번 반복하여 평균 성능을 도출한다.
