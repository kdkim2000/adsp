# 3과목 1장: R 기초와 데이터 마트

## 1. R 언어의 이해

### 1) R의 특징

* **오픈소스·무료**: 통계 계산과 그래픽을 위한 오픈소스 프로그래밍 언어이자 실행 환경으로, 누구나 무료로 사용할 수 있다.
* **CRAN(Comprehensive R Archive Network)**: 통계·머신러닝·시각화 등 다양한 목적의 패키지를 제공하는 저장소로, 필요한 패키지를 손쉽게 설치·확장할 수 있다.
* **벡터 기반 연산**: 반복문 없이도 벡터·행렬 단위로 한 번에 연산을 수행할 수 있어 대량 데이터 처리에 효율적이다.
* **인터프리터 언어**: 코드를 한 줄씩 즉시 실행하며 결과를 바로 확인할 수 있어 대화형 데이터 분석과 탐색에 적합하다.

### 2) R의 기본 데이터 구조

| 구조 | 설명 | 차원 |
|---|---|---|
| **벡터(Vector)** | 동일한 자료형의 1차원 데이터 집합 | 1차원 |
| **팩터(Factor)** | 범주형 데이터를 표현하는 자료형 (수준, Level 개념 포함) | 1차원 |
| **행렬(Matrix)** | 동일한 자료형의 2차원 데이터 구조 | 2차원 |
| **배열(Array)** | 동일한 자료형의 2차원 이상 데이터 구조 | 2차원 이상 |
| **데이터프레임(Data Frame)** | 열(column)마다 서로 다른 자료형을 가질 수 있는 표 형태의 구조 (가장 널리 사용) | 2차원 |
| **리스트(List)** | 서로 다른 자료형과 크기를 가진 객체들의 집합 | 제한 없음 |

> **핵심**: 데이터프레임은 R에서 가장 널리 활용되는 구조로, 각 열은 서로 다른 자료형을 가질 수 있으나 각 열 내부는 동일한 자료형이어야 한다. 행렬·배열은 모든 원소가 동일한 자료형이어야 한다는 점에서 데이터프레임과 구분된다.

## 2. 데이터 마트의 이해

### 1) 정의

데이터 마트(Data Mart)는 전사 데이터 웨어하우스로부터 특정 부서나 업무 영역의 요구에 맞게 추출·가공한 소규모 데이터 저장소이다.

### 2) 데이터 웨어하우스와의 관계

데이터 마트는 전사 데이터 웨어하우스의 부분집합으로, 특정 부서(예: 마케팅, 재무)나 특정 주제 영역의 분석 목적에 맞춰 설계된다. 전사 데이터 웨어하우스가 조직 전체의 통합 데이터를 다루는 반면, 데이터 마트는 특정 사용자 그룹이 빠르게 접근하여 활용할 수 있도록 범위를 좁혀 최적화한 저장소이다.

### 3) 데이터 웨어하우스(DW)의 4대 특성

데이터 웨어하우스의 개념을 처음 정립한 인몬(W. H. Inmon)은 데이터 웨어하우스가 갖추어야 할 특성을 다음 4가지로 정의했다.

| 특성 | 설명 |
|---|---|
| **주제 지향적(Subject-Oriented)** | 업무 프로세스가 아니라 고객·상품·거래와 같은 특정 주제(비즈니스 관심사)를 중심으로 데이터를 구성 |
| **통합성(Integrated)** | 여러 원천 시스템에서 수집된 데이터를 일관된 형식·명명 규칙·단위로 통합하여 저장 |
| **시계열성(Time-variant)** | 특정 시점의 스냅샷이 아니라, 시간의 흐름에 따라 변화하는 이력 데이터를 함께 저장하여 추세 분석이 가능 |
| **비휘발성(Non-volatile)** | 한 번 적재된 데이터는 임의로 수정·삭제되지 않고, 조회(Read) 위주로 안정적으로 유지됨 |

> **핵심**: 데이터 웨어하우스의 4대 특성은 "주제지향·통합성·시계열성·비휘발성"으로 암기한다. 운영 데이터베이스(OLTP)는 실시간으로 갱신·삭제되는 휘발성(Volatile) 데이터를 다루는 반면, 데이터 웨어하우스는 이력을 보존하는 비휘발성 데이터를 다룬다는 점이 대비되는 출제 포인트이다.

### 4) 데이터 마트 구축 프로세스

| 단계 | 내용 |
|---|---|
| **요건 정의** | 분석 목적에 따른 필요 데이터 항목 정의 |
| **데이터 추출·적재(ETL)** | 원천 시스템에서 데이터를 추출(Extract)하고, 분석 목적에 맞게 변환(Transform)한 뒤, 데이터 마트에 적재(Load) |
| **검증** | 데이터 정합성 및 품질 검증 |

## 3. 데이터 전처리

### 1) 결측값(Missing Value) 처리

데이터 분석에 앞서 결측값이 발생한 원인(메커니즘)을 파악하는 것이 처리 방법 선택의 출발점이다.

| 결측값 유형 | 의미 |
|---|---|
| **완전 무작위 결측(MCAR, Missing Completely At Random)** | 결측 발생이 다른 변수나 자기 자신의 값과 무관하게 완전히 무작위로 발생 |
| **무작위 결측(MAR, Missing At Random)** | 결측 발생이 다른 관측된 변수와는 관련이 있으나, 결측된 값 자체와는 관련이 없음 |
| **비무작위 결측(MNAR, Missing Not At Random)** | 결측 여부가 결측된 값 자체(응답하지 않은 이유가 그 값과 관련됨)에 의존 — 처리가 가장 까다로움 |

* **처리 방법**
  * **완전분석법(Complete Case Analysis)**: 결측치가 있는 행(케이스)을 통째로 제거. 구현이 간단하나 데이터 손실이 크고 편향이 발생할 수 있음
  * **단순 대치법(Single Imputation)**: 평균·중앙값·최빈값 등 대표값으로 결측치를 대체
  * **다중 대치법(Multiple Imputation)**: 결측치를 여러 번 그럴듯한 값으로 대체하여 여러 개의 완성된 데이터셋을 만든 뒤 결과를 통합 — 단순 대치법보다 추정의 불확실성을 더 현실적으로 반영함

> **핵심**: 결측값 유형(MCAR·MAR·MNAR)과 결측값 처리 방법(완전분석법·단순대치법·다중대치법)은 서로 다른 두 개의 분류 축이다 — 둘을 섞어 하나의 목록처럼 묻는 오답 선지에 주의한다.

### 2) 이상값(Outlier) 탐지

| 방법 | 판정 기준 |
|---|---|
| **IQR(사분위 범위) 기준** | 제1사분위수(Q1)와 제3사분위수(Q3)의 차이인 IQR = Q3 − Q1을 이용해, `Q1 − 1.5×IQR` 미만이거나 `Q3 + 1.5×IQR` 초과인 값을 이상값으로 판단 (박스플롯의 수염 밖 점) |
| **Z-score 기준** | 관측값을 표준화(평균 0, 표준편차 1)한 후, 절댓값이 특정 임계치(일반적으로 2 또는 3)를 초과하면 이상값으로 판단 |
| **시각화 기반 탐지** | 박스플롯(Boxplot), 산점도(Scatter Plot) 등을 통해 육안으로 극단값을 확인 |

### 3) 데이터 변환

| 기법 | 설명 |
|---|---|
| **정규화(Normalization, Min-Max)** | `(x - 최솟값) / (최댓값 - 최솟값)`로 계산하여 값의 범위를 0~1 사이로 조정 |
| **표준화(Standardization, Z-score)** | `(x - 평균) / 표준편차`로 계산하여 평균 0, 표준편차 1인 분포로 변환 |
| **로그 변환(Log Transformation)** | 왜도(Skewness)가 심하게 치우친 데이터에 로그를 취해 정규분포에 가깝게 조정 |
| **구간화(Binning)** | 연속형 변수를 몇 개의 구간(범주)으로 나누어 범주형 변수처럼 다루는 기법 |

### 4) 파생변수와 요약변수

* **파생변수(Derived Variable)**: 기존 변수를 조합·가공하여 새로운 의미를 갖도록 만든 변수 (예: 생년월일 → 나이, 소득과 부채 → 부채비율)
* **요약변수(Summary Variable)**: 여러 관측치의 정보를 하나의 대표값으로 요약하여 만든 변수 (예: 특정 기간의 총 구매금액, 평균 방문 주기)

> **핵심**: 데이터 전처리는 분석 결과의 신뢰성을 좌우하는 핵심 단계로, 전체 분석 과정에서 가장 많은 시간이 소요되는 작업 중 하나이다.
